MineTracking

MineTracking is a project I developed as part of a team for my Distributed Systems course during my Software Engineering studies. The idea originated from a real-world need identified within a mining company: to optimize operations and enable real-time monitoring of cargo transportation and traffic lights within a mine.

Since the project was linked to both Distributed Systems and Enterprise Architectures subjects, we designed a distributed system using a microservices-based architecture. Although our university primarily taught Java, my team and I had prior experience with other languages and frameworks, which allowed us to expand the project’s technological stack.

For the frontend, we used Vue.js, while the microservices were developed using both NestJS and ASP.NET. To facilitate communication between specific services and enable real-time data visualization, we implemented messaging brokers such as RabbitMQ and Mosquitto. Additionally, I developed an API Gateway using gRPC to connect the client with the various microservices, integrating an authentication service built with NestJS and JWT.

Key Features:

  • Microservices architecture
  • gRPC communication
  • JWT authentication service
  • Docker to deploy services
  • NestJS services
  • ASP.NET services
  • RabbitMQ and Mosquitto messaging brokers
  • Vue.js front end
  • Real-time data visualization using websockets

Technologies:

TypeScript C# Kotlin Node.js ASP.NET NestJS PostgreSQL TypeORM REST API Microservices Api Gateway Git GitHub Docker RabbitMQ Mosquitto Messaging Brokers WebSockets JWT Socket.io Vue.js HTML CSS

Domino Java Game

Domino Game in Java was a school project developed for my Software Architecture course. The objective was to build a local multiplayer Domino game that could be played over a network using TCP sockets.

What made this project particularly interesting was the requirement to design and implement our own software architecture from scratch. My team and I chose to implement the Broker Architecture, enabling multiple users on the same network to connect to a central broker via TCP sockets. This setup allowed players to exchange information and play the game in real time.

Key Features:

  • Broker Architecture Java implementation
  • Real-time data visualization
  • Local multiplayer mode
  • Complete domino game implementation
  • MVC implementation for the front end

Technologies:

Java Messaging Brokers TCP Sockets Git GitHub

Password Manager

Password Manager was my first project using Node.js after learning the basics and applying my backend knowledge gained at school. I started learning web development through a Udemy course, and this project was my first opportunity to put all that knowledge into practice.

The application features a simple vanilla frontend that interacts with an API I developed using Node.js and Express.js, with MongoDB as the database. All sensitive data, including passwords and user information, is securely encrypted within the system.

The idea for this project originated from my computer security class, where I learned that widely used password managers like Google’s might not be as secure as commonly believed. Instead of relying on third-party solutions, I decided to build my own password manager to better understand and control the security aspects involved.

Key Features:

  • JWT authentication system
  • Secure password storage
  • REST API with Express.js
  • Data encryption
  • Responsive dashboard
