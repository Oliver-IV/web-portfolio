MineTracking

MineTracking is a project that I did as a team for my Distributed Systems course in my Software Engineering career. It was born from a mining company's need to streamline its processes and to be able to monitor the transportation of cargo and traffic-lights within a mine in real time. As this project is part of my Distributed Systems subject, together with my Enterprise Architectures subject, I developed a Distributed System with a Microservices Based Architecture. At my university we had only worked with Java, but I along with the other members of my team, had already worked outside the school with other languages besides Java, and with different frameworks, so we decided to use Vue.js for the front end, NestJS and ASP.NET to develop the different microservices and Messaging Brokers like RabbitMQ and Mosquitto to achieve communication between some specific microservices, and real-time data visualization. In addition to all this, we made an API Gateway to communicate the client with the different microservices using gRPC, this used an uthentication service that uses JWT, both developed by me and using NestJS.

Key Features:

  • Microservices architecture
  • gRPC communication
  • JWT authentication service
  • Docker to deploy services
  • NestJS services
  • ASP.NET services
  • RabbitMQ and Mosquitto messaging brokers
  • Vue.js front end
  • Real-time data visualization using websockets

Technologies:

TypeScript C# Kotlin Node.js ASP.NET NestJS PostgreSQL TypeORM REST API Microservices Api Gateway Git GitHub Docker RabbitMQ Mosquitto Messaging Brokers WebSockets JWT Socket.io Vue.js HTML CSS

Domino Java Game

The Domino game in Java was a school project for my Software Architecture course. The purpose of the project was to make a Domino game that could be played in local multiplayer mode, using TCP Sockets for players that are connected to the same network. The interesting thing about this project was that we had to implement our own Software Architecture in Java, for which I and my team had to implement the Broker Architecture, with which we used TCP Sockets to achieve that 2 or more users connected at the same time to the Broker, could send information to each other and play the Domino game in real time.

Key Features:

  • Broker Architecture Java implementation
  • Real-time data visualization
  • Local multiplayer mode
  • Complete domino game implementation
  • MVC implementation for the front end

Technologies:

Java Messaging Brokers TCP Sockets Git GitHub

Password Manager

The Password Manager was my first project with Node.js after learning the basics and combining it with what I learned about back end in my school. I started learning web with a course in Udemy and I applied all my knowledge for the first time in this project. It consists of a simple vanilla front end that consumes an API that I developed with Node.js and Express.js, using MongoDB as a database and encrypting all the important data, such as passwords or users that are stored in this password manager. The idea came from the fact that in my computer security class I was told that using the google password manager was not as secure as we thought, so they recommended us to download a password manager, but something in me distrusted them, so I thought.... Why don't I make my own password manager?
